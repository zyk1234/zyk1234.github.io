centos7 mysql8 主从复制

  1. 修改mysql配置文件

    [mysqld]
log-bin=mysql-bin #开启二进制日志
server-id=1 #设置server-id
  2. 重启mysql，创建用于同步的用户账号
CREATE USER 'repl'@'192.168.0.34' identified with mysql_native_password by "456789";
GRANT REPLICATION SLAVE ON *.* TO 'repl'@'192.168.0.34';#分配权限
flush privileges;   #刷新权限
  1. 查看master状态，记录二进制文件名和位置
SHOW MASTER STATUS;
  1. 从服务器slave修改
[mysqld]
server-id=2 #设置server-id，必须唯一
  1. 重启mysql，打开mysql会话，执行同步SQL语句(需要主服务器主机名，登陆凭据，二进制文件的名称和位置)：
    CHANGE MASTER TO MASTER_HOST='192.168.0.33',MASTER_USER='rep1',MASTER_PASSWORD='456789',MASTER_LOG_FILE='mysql-bin.000001',MASTER_LOG_POS=73;
MASTER_LOG_FILE='mysql-bin.000001'    为第三步产生的结果
MASTER_LOG_POS=73;  第三部产生的结果
  1. 开启 
    start slave;
  2. 查看状态：
    show slave status\G;
    当状态中结果中的 Slave_IO_Running: Yes，并且 Slave_SQL_Running: Yes 时，主从复制成功

ubuntu mysql5主从

1.两台机器都安装mysql

apt install mysql-server

2.主服务器操作

修改配置文件

vim /etc/mysql/mysql.conf.d/mysqld.cnf 
bind-address = 0.0.0.0 #局域网可访问


server-id=1 #此选项是唯一的，每台机器不能重复
log_bin = /var/log/mysql/mysql-bin.log #启动日志，此选项为主从复制的主要内容

#binlog_do_db = 要主从的数据路
#binlog_ignore_db = 主从时忽略的数据库

创建用户：

mysql -uroot -p123456
create user repl; #创建用户
GRANT REPLICATION SLAVE ON *.* TO 'repl'@'172.168.20.206' IDENTIFIED BY '123456';
flush privileges;   #刷新权限


#查看日志
SHOW MASTER STATUS;
+------------------+----------+--------------+------------------+-------------------+
| File             | Position | Binlog_Do_DB | Binlog_Ignore_DB | Executed_Gtid_Set |
+------------------+----------+--------------+------------------+-------------------+
| mysql-bin.000002 |      154 |              |                  |                   |
+------------------+----------+--------------+------------------+-------------------+

重启mysql

service mysql restart

3. 从服务器操作

修改配置文件

vim /etc/mysql/mysql.conf.d/mysqld.cnf 
bind-address = 0.0.0.0 #局域网可访问

server-id=2

重启mysql

service mysql restart

连接master

stop slave;    #停止Slave    
change master to master_host='172.168.20.205',master_port=3306,master_user='repl',master_password='123456',master_log_file='mysql-bin.000002',master_log_pos=154; 

start slave;  #启动Slave

#查看slave状态
show slave status\G;
#Slave_IO_Running: Yes，并且 
#Slave_SQL_Running: Yes 时，主从复制成功
